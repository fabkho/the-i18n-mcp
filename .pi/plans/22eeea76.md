Refactor the-i18n-mcp to be CLI-first with MCP as an additional transport.

Architecture:
- `src/core/operations.ts` — all business logic as pure async functions returning plain objects
- `src/cli.ts` — CLI entry point with subcommands
- `src/server.ts` — thin MCP wrapper calling core operations
- `src/index.ts` — smart router (CLI if args present, MCP if stdio piped)
